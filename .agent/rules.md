# 目的
セッション中断・再開時におけるコンテキストの喪失を防ぎ、開発の連続性を自動化する。

# 制約事項
1. プロジェクトルートに `PROGRESS.md` を常設し、開発の単一情報源（SSOT）とする。
2. セッション開始時（再開時）は、ユーザーの初期プロンプトに関わらず、必ず最初に `PROGRESS.md` を読み込み、現在の進捗・文脈・次タスクを把握せよ。
3. タスク完了時、またはユーザーから「中断」の指示が出た際は、いかなる場合も直ちに `PROGRESS.md` を以下のフォーマットで上書き更新せよ。

# PROGRESS.md の記述フォーマット
- **Current Status**: 実装完了箇所（直近でコミット・保存した機能とファイル名）
- **Pending Tasks**: 残タスクのリスト（優先度順・即実行可能な粒度）
- **Known Issues**: 現在発生しているエラーや未解決の課題
- **Context**: 再開時にAIが引き継ぐべきアーキテクチャ上の決定事項や一時的な前提条件

# アクションの強制
再開時、`PROGRESS.md` の読み込みを完了し次第、「コンテキスト復元完了。次のタスク [Pending Tasksの最上位] の実装を開始しますか？」と一言のみ確認せよ。不要な挨拶は省くこと。

# 運用フロー
設定後、再開時のプロンプトは「再開」の2文字のみで完結する。AIが自動的に状況をロードし、次のコード生成に直結する仕組みとなる。
